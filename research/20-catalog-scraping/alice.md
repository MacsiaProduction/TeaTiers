# План скрапинга русскоязычных чайных сайтов для каталога TeaTiers: источники, инструменты, правовые рамки и рекомендации

## Главные выводы

Исследование выявило, что **artoftea.ru является единственным приоритетным источником** для скрапинга фактографических данных о чае (названия, тип, регион происхождения, страна). Его robots.txt разрешает доступ к страницам товаров, сайт построен на серверном рендеринге (OpenCart) и содержит хорошо структурированные данные для ~200–300 позиций [artoftea.ru](https://artoftea.ru/robots.txt)[artoftea.ru](https://artoftea.ru/oolong/da-hun-pao-kremovyj). **tea.ru полностью не пригоден** для скрапинга: robots.txt запрещает /product/ и /products/ для всех пользовательских агентов [tea.ru](https://tea.ru/robots.txt). Маркетплейсы (Ozon, Wildberries) технически сложны (JS-рендеринг, антибот-защита) и несут повышенные юридические риски [www.ozon.ru](https://www.ozon.ru/robots.txt)[www.wildberries.ru](https://www.wildberries.ru/robots.txt)[parsingmaster.com](https://parsingmaster.com/blog-post/legalnost-parsinga-v-2026-yuridicheskiy-gayd/). Российское законодательство (ст. 1334 ГК РФ о правах изготовителя базы данных и ст. 1270 ГК РФ об авторских правах) требует ограничиться сбором **фактов**, не включая в публичный APK скопированные описания и изображения [www.consultant.ru](https://www.consultant.ru/document/cons_doc_LAW_64629/084d75757a923474029e166de8b00d3150994bfa/)[www.consultant.ru](https://www.consultant.ru/document/cons_doc_LAW_64629/4db5f620672d686c1a7ba12eafa85bfa6dcd2296/). Рекомендуемый стек: `httpx` 0.28.1 + `selectolax` 0.4.0 + `reppy` для соблюдения `robots.txt` [generalistprogrammer.com](https://generalistprogrammer.com/tutorials/httpx-python-package-guide)[generalistprogrammer.com](https://generalistprogrammer.com/tutorials/selectolax-python-package-guide)[github.com](https://github.com/seomoz/reppy). Итоговая рекомендация: однократный bulk-импорт с artoftea.ru (категории чая → страницы продуктов → фактографические поля → Postgres через существующий seed-путь с дедупликацией по ключу `tea.name_norm` + pg_trgm), после чего LLM-обогащение генерирует собственные описания.

## 1. Ранжирование источников

### 1.1. artoftea.ru (приоритет №1)

**robots.txt:** продукт не запрещён. Заблокированы только служебные пути: `/*route=account/`, `/*route=checkout/`, `/*route=product/search`, сортировка, поиск по параметрам [artoftea.ru](https://artoftea.ru/robots.txt). Категории товаров и страницы отдельных продуктов **доступны для индексации**.

**Техническая структура:** OpenCart, серверный рендеринг. URL-схема: `artoftea.ru/{тип_чая}/{slug-название}`. Категории: `/oolong/`, `/puer/`, `/green/`, `/red/` и т.д. [artoftea.ru](https://artoftea.ru/oolong/da-hun-pao-kremovyj). На странице продукта присутствуют:
- Название (RU) — например, «Да Хун Пао «Кремовый»».
- Тип: «улун».
- Регион: «горный хребет Уишань, провинция Фуцзянь».
- Год сбора (например, 2025).
- Страна производства: «Китай».
- Формат: листовой.
- Аромат и вкус (описательные тексты — **подлежат отдельной обработке, см. раздел 4**).

**Качество данных:** высокая согласованность таксономии. Категории чётко разделены по типу чая. Оценочный размер чайного каталога: 200–300 позиций [artoftea.ru](https://artoftea.ru/oolong/da-hun-pao-kremovyj).

### 1.2. tea.ru (не рекомендуется)

**robots.txt:** `/product/` и `/products/` **запрещены** для всех пользовательских агентов, включая Yandex и Googlebot [tea.ru](https://tea.ru/robots.txt). Скрапинг этих путей является прямым нарушением протокола `robots.txt`. Каталог формально богат (тысячи позиций), но юридически и этически использовать его как источник недопустимо.

### 1.3. GorkovChay.ru (ограниченно пригоден)

**robots.txt:** OpenCart-реализация с очень жёсткими ограничениями: `Disallow: *?*` — запрещены почти все URL с параметрами запроса. `Disallow: *.html` — запрещены html-страницы [gorkovchay.ru](https://gorkovchay.ru/robots.txt). Каталог (около 1000 позиций по утверждению сайта [gorkovchay.ru](https://gorkovchay.ru/)) практически полностью блокирован. **Не рекомендуется** в качестве первичного источника.

### 1.4. Ozon.ru и Wildberries.ru (высокий риск / низкое качество)

**robots.txt:** Ozon не запрещает `/product/` напрямую [www.ozon.ru](https://www.ozon.ru/robots.txt); Wildberries разрешает `/catalog/*` [www.wildberries.ru](https://www.wildberries.ru/robots.txt). Однако оба сайта имеют:
- **Антибот-защиту:** CAPTCHA, Cloudflare, JS-рендеринг. Wb использует шаблонизацию Mustache.js (вёрстка видна, но часть контента динамическая), Ozon — React [www.ozon.ru](https://www.ozon.ru/product/chaiy-te-graph-ulkun-ulun-100-g-247618868/)[www.ozon.ru](https://www.ozon.ru/product/chay-da-hun-pao-bolshoy-krasnyy-halat-premium-ulun-chaynaya-masterskaya-100g-kitayskiy-170454728/)[www.wildberries.ru](https://www.wildberries.ru/catalog/166766204/detail.aspx).
- **Низкое качество фактов:** один и тот же чай продаётся разными продавцами с противоречивыми атрибутами. Страна производства часто указана как «Россия» (страна фасовки), а не фактического происхождения [www.ozon.ru](https://www.ozon.ru/product/chay-da-hun-pao-bolshoy-krasnyy-halat-premium-ulun-chaynaya-masterskaya-100g-kitayskiy-170454728/)[www.wildberries.ru](https://www.wildberries.ru/catalog/166766204/detail.aspx).
- **Юридические риски:** см. раздел 4.

**Применимость:** только для точечной верификации названий и розничных цен — **не для систематического пополнения каталога**.

### 1.5. Русскоязычные чайные энциклопедии

**Вывод: специализированная структурированная база отсутствует.** Источники ограничены: общие статьи в Wikipedia [ru.wikipedia.org](https://ru.wikipedia.org/wiki/%D0%A7%D0%B0%D0%B9_%D0%B2_%D0%A0%D0%BE%D1%81%D1%81%D0%B8%D0%B8), база сортов teaepicure.com (на английском, не ориентирована на РФ) [teaepicure.com](https://teaepicure.com/cultivar-database/), и блог realchinatea.ru [realchinatea.ru](https://realchinatea.ru/tpost/v4omn1fpk1-sem-grupp-kitaiskih-chaev) без структурированных данных. Wikidata (упомянутый в контексте проекта как уровень enrichment) остаётся лучшим открытым структурированным источником для фактов о типе, происхождении и сорте чая — но он **не русскоязычный** и не дает русских названий прямым scraping-доступом.

## 2. Стек инструментов скрапинга

| Инструмент | Версия | Лицензия | Назначение |
|---|---|---|---|
| `httpx` | 0.28.1 | BSD-3-Clause [generalistprogrammer.com](https://generalistprogrammer.com/tutorials/httpx-python-package-guide) | HTTP-клиент для получения HTML-страниц. Поддержка HTTP/2, повторных попыток. |
| `selectolax` | 0.4.0 | MIT [generalistprogrammer.com](https://generalistprogrammer.com/tutorials/selectolax-python-package-guide) | Быстрый парсинг HTML с поддержкой CSS-селекторов. Альтернатива BeautifulSoup/LXML. |
| `reppy` | 0.5.x | MIT [github.com](https://github.com/seomoz/reppy) | Правильная загрузка и парсинг `robots.txt`, проверка разрешённости URL для заданного User-Agent. |

**Не рекомендуется:** `Scrapy` (избыточен для однократного импорта ~200–300 страниц), `Playwright` (не требуется — artoftea.ru не использует JS-рендеринг для данных продукта). `urllib.robotparser` из стандартной библиотеки — альтернатива, но `reppy` даёт более точную поддержку современных директив (Crawl-Delay, Sitemaps).

**Политика вежливости:**
- User-Agent: `TeaTiers-bot/1.0 (educational project; scraping fact data; contact: project@teatiers.local)`
- Задержка между запросами: 1–2 секунды (реализуется `time.sleep(random.uniform(1, 2))`).
- Кеширование: кеш HTTP-ответов на диске (через `diskcache` или аналоги) для возможности инкрементального обновления без повторного скачивания.
- Обязательная проверка `robots.txt` через `reppy` перед каждым целевым URL.

## 3. Модель данных и дедупликация

### 3.1. Маппинг полей artoftea.ru → Postgres

| Поле на сайте | Таблица / колонка | Примечание |
|---|---|---|
| Название (RU) — заголовок страницы | `tea_name` (locale = 'ru', is_primary = true) | Извлекается из `<h1>` или мета-структуры. |
| Тип (например, «улун») | `tea.type` | Маппинг на константу (en): oolong, puer, green, red, white, hei cha, etc. |
| Регион сбора | `tea.region` | Например, «горный хребет Уишань, провинция Фуцзянь». |
| Страна производства | `tea.origin_country` | Например, «Китай». |
| Год сбора | — | Может быть добавлен как атрибут в `tea_name` или как мета-поле. |
| Культивар / сорт | `tea.cultivar` | Если явно указан (например, «Тегуаньинь»). |

### 3.2. Дедупликация

Используется существующий ключ `tea.name_norm` + pg_trgm GIN-индекс:

1. При импорте для каждого нового названия генерируется `name_norm` (нижний регистр, удаление диакритики/кавычек, замена кириллических вариантов на латинские аналоги для известных названий: «Да Хун Пао» → `da hong pao`).
2. Выполняется поиск pg_trgm по `tea_name.name_norm` с порогом > 0.6. Если совпадение найдено — запись **не создаётся заново**, а обновляются поля более надёжного источника (по правилу: Wikidata > artoftea.ru > miss-log).
3. Для мультиязычных названий: если `tea` уже существует, вставляется новая запись в `tea_name` с locale='ru' и source='artoftea.ru'.
4. **Идемпотентность:** импорт использует `ON CONFLICT (tea_id, locale, source_url) DO UPDATE` — повторный запуск не создаёт дубликатов.

### 3.3. Конфликт полей

Если artoftea.ru и Wikidata дают разные `origin_country` для одного чая — приоритет у Wikidata. Если несовпадение в `type` — запись маркируется `verified = false` и выводится оператору на верификацию.

## 4. Факты vs. охраняемое авторским правом выражение

### 4.1. Политика

**Безопасно хранить и распространять в APK (факты):**
- Названия чаёв на русском, китайском (zh-Hans, pinyin) — низкий риск.
- Тип, страна, регион, сорт, культивар.
- Год сбора (факт, не творческое выражение).

**Не включать в публичный APK:**
- **Описания вкуса/аромата/воздействия** — это творческий текст, охраняемый авторским правом (ст. 1270 ГК РФ). На сайте artoftea.ru они присутствуют, например: «Тёплый, сливочно-карамельный, кондитерский…» [artoftea.ru](https://artoftea.ru/oolong/da-hun-pao-kremovyj).
- **Изображения** продуктов.
- Отзывы пользователей.

**Правило:**
- Скрапинг собирает факты → они сохраняются в `tea`/`tea_name`/`tea.flavor` только если извлекаются **через структурированный маппинг** (а не копированием текста).
- Сырой HTML-текст описаний может временно храниться **только на сервере** как входные данные для LLM-обогащения (поле `source_text`), но **не экспортируется в публичный APK**.
- Существующая LLM-генерация (YandexGPT) создаёт собственные описания — именно они идут в `tea_description`.

### 4.2. Юридические рамки

**Российское законодательство:**
- **Авторское право** (ст. 1270, 1301 ГК РФ) — копирование текстов описаний и фото без разрешения может повлечь компенсацию до 10 млн руб. [parsingmaster.com](https://parsingmaster.com/blog-post/legalnost-parsinga-v-2026-yuridicheskiy-gayd/).
- **Право изготовителя базы данных** (ст. 1333–1335 ГК РФ) — извлечение и повторное использование существенной части базы данных artoftea.ru без согласия может нарушать права изготовителя (сайт как база данных). **Риск снижается**, если извлекаются только факты (названия, типы, страны), а не весь контент товаров [www.consultant.ru](https://www.consultant.ru/document/cons_doc_LAW_64629/084d75757a923474029e166de8b00d3150994bfa/)[www.consultant.ru](https://www.consultant.ru/document/cons_doc_LAW_64629/4db5f620672d686c1a7ba12eafa85bfa6dcd2296/).
- **152-ФЗ / персональные данные:** не релевантны, если не собираются данные о пользователях. С 30 мая 2025 года ужесточены штрафы за утечку ПДн [nalog-nalog.ru](https://nalog-nalog.ru/personalnye_dannye/shtrafy-roskomnadzora-za-personalnye-dannye-v-2025-godu/), но данные чаёв к ним не относятся.

**Практическое правило для персонального проекта TeaTiers:**
- Скрапинг публичных страниц artoftea.ru без обхода CAPTCHA/авторизации — **серый, но не криминальный** сценарий, если не копировать охраняемые тексты.
- **Минимизировать риск:** не превышать разумной нагрузки, соблюдать `robots.txt`, указать User-Agent, кешировать, не републиковать скопированные описания. При публичном судебном споре (маловероятно для персонального проекта) суд будет оценивать как факт добросовестное использование и некоммерческий характер.

## 5. Операционная модель: однократный bulk-импорт

### 5.1. Почему не постоянный скрапинг

- artoftea.ru — живой магазин; частота изменений карточки товара (название, тип, регион) низкая. Цены и остатки меняются, но они **не нужны** для каталога (TeaTiers — не ценовой агрегатор).
- Поддерживать постоянно работающий краулер на 4GB VM нецелесообразно из-за нагрузки и риска блокировки.
- **Рекомендация:** однократный запуск скрапера локально (на машине разработчика), экспорт в CSV/JSON, затем импорт в Postgres через существующий seed-путь.

### 5.2. Приоритет по miss-log

После первичного импорта следует:
1. Загрузить существующую таблицу `catalog_miss` (неразрешённые запросы от `/resolve`).
2. Сопоставить miss-запросы с названиями, полученными с artoftea.ru.
3. Если чай из miss-log найден — импортировать его **с verified = false** и передать на LLM-обогащение.
4. Если не найден — оставить в miss-log для ручного добавления.

### 5.3. Интеграция с enrichment

После импорта фактов в `tea`, `tea_name`:
- Запустить существующий enrichment-пайплайн: Wikidata → YandexGPT.
- LLM получает фактические данные + `source_text` (если сохранён) и генерирует flavour+RU blurb.
- Сгенерированные описания сохраняются в `tea_description` (locale='ru', source='YandexGPT').

## 6. Итоговая рекомендация и последовательность действий

**Что строить в первую очередь (конкретный first cut):**

1. **Источник:** `artoftea.ru` — единственный на данный момент.
2. **Стек:** Python + `httpx` 0.28.1 + `selectolax` 0.4.0 + `reppy` (или `urllib.robotparser`).
3. **Процесс:**
   - Получить список категорий: `/oolong/`, `/puer/`, `/green/`, `/red/`, `/white/`, `/gaba/` и др. через элементы навигации или sitemap.
   - Для каждой категории выполнить пагинацию (заметили «Показано с 1 по 12 из 717» и ссылки на страницы) [artoftea.ru](https://artoftea.ru/index.php?route=product/category&path=65).
   - Для каждой карточки товара из списка перейти на страницу продукта.
   - Извлечь **только фактические поля**: название, тип, регион, страна, культивар, год сбора.
   - **Не сохранять** verbatim описа́ния, отзывы, изображения.
   - Экспортировать в JSON или CSV.
4. **Импорт:**
   - Загрузить в Postgres через существующий seed-скрипт с `ON CONFLICT` и дедупликацией по `name_norm`.
   - Пометка: `source = 'artoftea.ru'`, `verified = false`.
5. **Enrichment:**
   - Запустить Wikidata-запросы для уточнения типа/происхождения.
   - Если Wikidata не дал результата — запустить YandexGPT с фактами и `source_text`.
   - **GPT генерирует оригинальное описание** — оно идёт в `tea_description`.
6. **QA:**
   - Вручную проверить 10–15 импортированных записей.
   - Если качество приемлемо — снять `verified = true` для всех или части.

**Важные ограничения:**
- artoftea.ru не содержит китайских (zh-Hans, pinyin) названий в явном виде. Их придётся добирать из Wikidata или через LLM-генерацию.
- Сайт не различает сорт и культивар последовательно — потребуется бизнес-правило для маппинга.
- **tea.ru не трогать** — robots.txt не позволяет.
- Маркетплейсы (Ozon, WB) использовать только для сверки цен/названий, но не как источник для каталога.
