# Рекомендации

- **MVP:** MapLibre + OpenStreetMap (без ключа). Причина: не требует API-ключей или GMS, свободно работает в RuStore-сборке, прост в старте. (Картографию можно брать с общедоступных тайлов OSM или бесплатных провайдеров, обратный геокодинг – публичным Nominatim/Photon с простым кэшированием.)  
- **Долгосрочно:** Yandex MapKit (full-версия). Причина: локализация на РФ, собственный обратный геокодер Yandex (который даёт названия на русском и адаптирован под Россию), хорошая интеграция MapKit в Compose через AndroidView. Привлекательно для России, хотя требует Yandex-ключ/условий. 
(В крайнем случае Google-Maps-композицию можно опционально включить, но без GMS она бесполезна.)

# Интерфейс абстракции на Kotlin

Например, можно задать такой интерфейс:

```kotlin
interface MapProvider {
    /** Показывает карту с возможностью выбора (перетаскивания/нажатия) точки. */
    @Composable
    fun LocationPicker(
        initialLocation: LatLng,
        onLocationSelected: (LatLng) -> Unit
    )

    /** Обратное геокодирование координат в человекочитаемый адрес (на русском). */
    suspend fun reverseGeocode(latLng: LatLng, locale: Locale = Locale("ru")): String?
}
```

- Для **MapLibre** реализация: `LocationPicker` использует `MapLibreCompose.MapView` (или `AndroidView { MapView }`) и возвращает `LatLng`. `reverseGeocode` – вызывает Nominatim/Photon REST API.  
- Для **Yandex**: `LocationPicker` – AndroidView с `com.yandex.mapkit.mapview.MapView`, настройки подобно примеру с Draggable Placemark; `reverseGeocode` – через встроенный Yandex Geocoder API (класс `SearchManager`, `GeocodeRequest`).  
- (Для Google: `@Composable GoogleMap { ... }` + Geocoder API, если когда-нибудь потребуется.)

# Поток разрешений

- **Местооположение (ACCESS_FINE/COARSE):** **Не нужен** для выбора точки по карте! Если пользователь просто тыкает или драггирует пин, разрешения не требуется.  
- **Когда запрашивать:** только при попытке использовать «моя локация» (текущий GPS). Если в приложении есть кнопка «Использовать моё местоположение», тогда при её нажатии нужно проверить и запросить `ACCESS_COARSE_LOCATION` или `ACCESS_FINE_LOCATION` согласно требуемой точности.  
- **Совет:** Запрашивать в момент необходимости с объяснением. Если пользователь просто ставит пин на карту, никаких разрешений не брать.  
- **Хранение:** Сохраняйте только конечные координаты выбранной точки (LatLng). Не записывайте лишние данные геолокации или историю пользователя.

# Офлайн-возможности

- **Yandex MapKit:** в бесплатной версии офлайн-карты не работают. Даже кэшированные тайлы недоступны для бесплатного ключа. Обратное геокодирование – только по сети. То есть без интернета карта не загрузится, поиск/адрес не сработает.  
- **MapLibre + OSM:** можно настроить офлайн: загрузить стили и данные (использовать [OfflineRegion](https://docs.maplibre.org/android/overview/offline/)), кэшировать тайлы. Тогда карта будет рендериться без сети (ограниченная область). Обратный геокодер (Nominatim/Photon) **тоже нужен онлайн** – без интернета адреса не получить (можно лишь кэшировать редкие запросы вручную).  
- **Google Maps API:** стандартный SDK не поддерживает офлайн. Android Geocoder работает на внутреннем движке Google, но на большинстве устройств требует интернет; при его отсутствии обычно возвращает пустой результат.  
- **Android Geocoder:** системный класс `android.location.Geocoder` под капотом вызывает сетевые сервисы и **«не всегда может работать офлайн»** (часть данных может быть закеширована, но это непредсказуемо). Поэтому полагаться на него без сети не стоит.

# Источники

- Yandex MapKit SDK docs: gradle-координаты и получение ключа. Бесплатный план – до 1000 DAU с условиями (open app).  
- Yandex MapKit лицензии: без офлайна в бесплатной версии.  
- MapLibre Android SDK (имплементация `org.maplibre.gl:android-sdk`, версия 11.x); MapLibre Compose (группа `org.maplibre.compose:maplibre-compose:0.13.0`).  
- Политика OpenStreetMap по тайлам: бесплатное использование с 1 req/sec и обязательным кэшированием.  
- Nominatim (OSM) Usage Policy: публичный API разрешает максимум ~1 запрос/сек (в сумме по приложению).  
- Google Maps Compose (6.12.0) docs: требует GMS и API-ключ (биллинг, $200 в мес.).  
- Android-разрешения: официально, получение местоположения требуется только при GPS-таргетинге (не для простого выбора точки). См. [Android Guide по Permissions](https://developer.android.com/guide/topics/permissions/overview) (упрощённо).  
- Конфиденциальность: храните только LatLng, без привязки к пользователю.

**Неуверенные моменты:** Yandex Geocoder API ограничения (суточные лимиты) официально не опубликованы (в сообществе упоминают ~25000 запросов/день, но это не подтверждено). Также неизвестны конкретные ограничения в RuStore-билде (политика Yandex не упоминает ограничений по дистрибуции). Бесплатный Tile-сервис OSM не регламентирует чёткий лимит запросов, лишь рекомендует кэшировать (предполагается разумный трафик). Возможности Photon API безлимитны, но надёжность публичного сервера не гарантирована (для серьёзной нагрузки лучше свой инстанс).
