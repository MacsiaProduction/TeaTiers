# Вердикт по §6 плана

**«Расширить: добавить веб-запрос как fallback»** — доступен работающий вариант. Из всех рассмотренных сервисов наиболее подходящим оказался **Google Gemini API** (режим с `google_search`): он бесплатен (500 запросов/день поиска на Gemini 2.5), допускает хранение результатов и даёт качественные ответы на ru/zh-запросы. Без VPN (на случай недоступности Gemini) можно использовать YandexGPT как резерв.

**Если обновляем план §6:** предлагается первичным провайдером **Google Gemini (gemini-2.5-flash)** с включенным инструментом поиска Google. Запрос должен быть вида:

```json
POST https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent
Content-Type: application/json
Authorization: Bearer $API_KEY

{
  "prompt": "Найдите фактологическую информацию о чае по названию и выведите JSON {name: {ru, en, zh, pinyin}, type, origin, blurb, flavor_profile} согласно схеме.",
  "temperature": 0.2,
  "max_output_tokens": 512,
  "tools": [
    { "google_search": {} }
  ]
}
```

Выходной ответ должен быть строго в заранее заданной JSON-схеме (включая поля «name», «type», «origin», «blurb», «flavor_profile»). Нужно отключить обучение (training) на пользовательских запросах (`trainingMode=off`). При недоступности Gemini (например, ограничение региона) использовать резервно **Claude с web_search** или YandexGPT (без VPN).
