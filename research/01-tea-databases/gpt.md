## Рекомендуемая стратегия

- **Использовать открытые источники.** В первую очередь зеркалировать **Wikidata** и **Wikipedia (en/ru/zh)**, так как это свободно распространяемые (CC0/CC-BY-SA) и регулярно обновляемые БД. Wikidata предоставляет структурированные записи о сортах и видах чая (в том числе со свойствами «родина/регион», «тип», «культивара» и т.д.); русский и китайский языковые метки к элементам есть по возможности. Wikipedia в разных языках содержит статьи о многих чаях; её данные можно легально кешировать с указанием лицензии (CC-BY-SA 4.0). Можно периодически (раз в неделю/месяц) обновлять дампами или API (MediaWiki). Эти источники дают базовые наименования и факты (хотя объём специализированных полей ограничен).

- **Открытые каталоги продуктов.** Рассмотреть **Open Food Facts** для получения информации о брендах/марках чаёв и их локализованных названиях. OFF – общедоступный проект (ODbL) с миллионами записей продуктов (в т.ч. чаёв). Данные OFF можно скачаивать или запрашивать по API, но при использовании нужно соблюдать лицензию (указать источник, share-alike для производных). Русские/китайские названия в OFF обычно появляются через пользовательские переводы, поэтому покрытие будет неполным.

- **Исключить проприетарные сервисы.** **Steepster**, **RateTea**, **TeaDB/MyTeaDB**, **Tea Guardian**, **World Tea Directory** и подобные сервисы – авторские сайты, не предоставляют открытой лицензии и запрещают копирование/сканирование. Их данные нельзя свободно зеркалить или переиспользовать. То же касается китайской Baidu Baike (закрытая лицензия) и любительских каталогов без явного права на передачу данных.

- **Kaggle и другие наборы данных.** Отдельные датасеты на Kaggle по чаю имеются, но их лицензии часто ограничивают коммерческое использование (например, CC BY-NC-SA) или не указаны. Поэтому полагаться на них для базы данных в коммерческом приложении рискованно. Можно использовать подобные данные скорее для анализа, но не как основной источник имен и метаданных чаёв.

- **Семена-список и вручную.** Если после импорта открытых данных каталог остаётся бедным (особенно на русском/китайском), следует сформировать **качественный набор из ~200–500 наиболее распространённых сортов**. Это включит китайские («十大名茶», «白茶» и т.д.), индийские (Дарджилинг, Ассам и др.), японские (Сэнтя, Маття), российские (ультраферментированные ферментированные чаи и т.д.) и т.д. Для каждого надо найти авторитетные эталонные названия на русском и китайском. Например, список «Десять знаменитых чаёв Китая» может дать начальный набор; русские и китайские Википедии содержат много готовых переводов (например, см. «Лунцзин/龙井茶»). Реальная ручная проверка этого количества сортов может занять несколько недель работы одного специалиста, но создаст необходимую основу для локализации.

## Предлагаемая схема PostgreSQL

Минимально требуется две связанные таблицы: одна – основная информация о чае, вторая – локализованные названия. Например:

```sql
CREATE TABLE tea (
    id SERIAL PRIMARY KEY,
    type    TEXT,          -- тип чая (зелёный, улун, пуэр и т.д.)
    origin  TEXT,          -- страна/регион происхождения
    cultivar TEXT,         -- сорт/культивара
    oxidation TEXT,        -- степень ферментации (лёгкая, полная и т.д.)
    vendor  TEXT           -- бренд или производитель
    -- дополнительные поля: описание, вкус, заметки и т.п.
);

CREATE TABLE tea_name (
    tea_id   INT REFERENCES tea(id),
    locale   VARCHAR(5),   -- код языка (e.g. 'ru', 'zh', 'en')
    name     TEXT,
    is_primary BOOLEAN,    -- является ли основным названием
    PRIMARY KEY (tea_id, locale, name)
);
```

Для быстрого поиска по частичному совпадению названий добавьте индекс с поддержкой n-gram или full-text. Например, расширение `pg_trgm` позволяет создать GIN-индекс:

```sql
CREATE INDEX tea_name_name_trgm ON tea_name USING GIN (name gin_trgm_ops);
```

Или используйте `to_tsvector()` с конфигурацией языка (для русского и китайского отдельно) и GIN-индексы для полнотекстового поиска. Это ускорит префиксный/подстрочный поиск по названиям на кириллице и иероглифах. Также стоит индексировать `locale` и `is_primary` в tea_name для фильтрации языка и основного имени.

## Источники

- Steepster – **Terms of Use** (раздел условий о копировании): *«You may not copy, reproduce, distribute... mirror any material contained on the Site»* (последняя проверка 2026).  
- Wikidata – **Wikidata:Licensing** (официальная политика): данные (структурированные) выложены под CC0 (публидомен) (последняя проверка 2026).  
- Wikipedia – например, страница категории «Tea» (англ.) с лицензией: *«Text is available under the Creative Commons Attribution-ShareAlike 4.0 License»*. Лицензия CC BY-SA 4.0 применяется ко всем статьям (рус., кит. версии аналогично).  
- Open Food Facts – статья Википедии про OFF: *«OFF licensed under the Open Database License, while its artwork... CC BY-SA»* (последняя проверка 2026).  
- Пример перевода названий – страница «Знаменитые чаи Китая» (рус.), таблица 10 популярных чаёв с китайскими оригиналами и транскрипцией.  

## Неопределённости

- **Российские базы чаёв:** не найдено свободно доступных специализированных каталогов чаёв на русском. Скорее всего, любые существующие «чайные порталы» не позволяют репликацию данных.  
- **Китайские закрытые базы:** Baidu Baike и подобные энциклопедии – не открыты для копирования. Официальные «каталоги чаёв» в Китае существуют (например, государственные классификации), но их лицензии и формат данных неизвестны.  
- **Kaggle-датасеты:** отдельные наборы о чае часто имеют ограничения (NC) и нестабильны. Их лицензии нужно проверять индивидуально.  
- **Количество записей в Wikidata:** точное число записей, связанных с чаем (сорты, бренды), непонятно; может потребоваться SPARQL-запрос для уточнения.  
- **Локализация названий:** авторитетность некоторых переводов сомнительна (например, транскрипция китайских имён). Требуется экспертная проверка вручную.
